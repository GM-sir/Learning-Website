Forms and Input

Form Tag (<form>) The <form> tag is used to collect user input and send it to a server.

Code Example: 

     <form action="submit_form.php" method="post"> 
     <label for="name">Name:</label> 
     <input type="text" id="name" name="name"><br><br>
     <label for="password">Password:</label> 
     <input type="password" id="password" name="password"><br><br>
     <label for="email">Email:</label> 
     <input type="email" id="email" name="email"><br><br>
     <input type="submit" value="Submit">     
     </form>
Input Types
  1. text: Single-line text input.

  2. password: Obscured input for passwords.

  3. email: For email addresses with validation.

  4. submit: Button to submit the form.
Textareas and Buttons
  • <textarea>: Multi-line text input.

  • <button>: Customizable button.
Code Example: 
    <form> 
      <label for="message">Message:
 
      <textarea id="message" name="message" rows="4" cols="50">

 
      <button type="button" onclick="alert('Button clicked!')">Click Me 
    </form>
HTML provides built-in form validation features that can be applied using attributes without requiring JavaScript. Below is a comprehensive explanation and example of form validation using HTML:

HTML Form Validation Attributes

  1. required: required: Ensures the field is filled before submission.

  2. pattern: Specifies a regular expression for custom validation

  3. maxlength and minlength: Restricts the number of characters.

  4. type: Defines input types with specific validation (e.g., email, number).

  5. min and max: Sets a range for numeric inputs.

  6. step: Specifies intervals for numeric inputs.

  7. title: Provides a tooltip with instructions for validation.

Example: Validating a Form 
    <!DOCTYPE html>
    <html lang="en"> 
    <head> 
      <meta charset="UTF-8">
      <meta name="viewport" content="width=device-width, initial-scale=1.0"> 
      <title>Form Validation</title> 
      </head> 
      <body>
        <form action="submit_form.php" method="post"> 
          <!-- Name Field (Required, Min Length: 3, Max Length: 20) --> 
          <label for="name">Name:</label>
          <input type="text" id="name" name="name" required minlength="3" maxlength="20" title="Name must be between 3 and 20 characters."> <br><br>
          <!-- Email Field (Required, Valid Email Format) --> 
          <label for="email">Email:</label>
          <input type="email" id="email" name="email" required title="Please enter a valid email address."> <br><br>
          <!-- Password Field (Required, Min Length: 6) --> 
          <label for="password">Password:</label>
          <input type="password" id="password" name="password" required minlength="6" title="Password must be at least 6 characters long."> 


          <!-- Age Field (Optional, Numeric, Between 18 and 100) --> 
          <label for="age">Age:</label>
          <input type="number" id="age" name="age" min="18" max="100" step="1" title="Age must be between 18 and 100."> <br><br>
          <!-- Phone Number Field (Required, Custom Pattern) --> 
          <label for="phone">Phone Number:</label>
          <input type="tel" id="phone" name="phone" required pattern="[0-9]{10}" title="Phone number must be 10 digits."> <br><br> 
          <!-- Submit Button --> 
          <input type="submit" value="Submit">          
        </form>
      </body>
    </html>
Explanation of Attributes in the Example
  1. Name Field:
    • The required attribute ensures the user cannot leave it blank.

    • minlength and maxlength enforce length restrictions.

    • title provides user guidance.


  2. E-Mail Field:
    • The type="email" validates the input to match an email format (e.g., example@domain.com).

  3. Password Field:
    • The minlength="6" ensures a minimum password length.

  4. Age Field:
    • type="number" ensures only numbers are allowed.

    • min and max set the valid age range.

    • step="1" ensures the input increases or decreases by 1 unit.


  5. Phone Number Field:
    • type="tel" is used for phone numbers.

    • pattern="[0-9]{10}" enforces exactly 10 numeric digits.


How HTML Validation Works

  • If a user submits invalid data, the browser will block the submission and display error messages.

  • For example, if the email format is incorrect, the browser shows: "Please include an '@' in the email address."
